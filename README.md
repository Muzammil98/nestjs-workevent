Scope and Due Date
Create a calendar API in NestJS and Postgres (typeorm). Due Date is Monday June 3rd, 2024. You will present the project to the HealthHelper engineers on this date.

Description
At HealthHelper we have users. We need to know those users availability (the days and hours that those users will be working). So this project is a basic calendar implementation (backend only).

Each user will have id, firstName, lastName
API - add API to add workevent for the user. Let’s make this API payload to be an array of workevents and anything else that you think is necessary.
Each work event is 15 minutes long, so if user is working from 8am to 10am, that would be 8 15 minute workevents
Each workevent has id, startsOn , endsOn, createdOn and the relationship to the user. Each user can have many workevents but each workevent can only belong to one user.
API - get workevents for many users. The api will take in from and to and will return workevents for all users during the time between from and to.
The only requirements are that the API is done in NestJS and the database used to store workevents is Postgres. Everything else is up to your imagination.
OPTIONAL - create a calendar UI that will show the workevents on the UI. Use https://www.npmjs.com/package/react-big-calendar or similar library.
OPTIONAL - present deployment of this project to Kubernetes on AWS, what services in AWS will you need, how will you configure, what about CICD?
